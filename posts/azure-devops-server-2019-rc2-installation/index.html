Azure DevOps Server 2019 RC2 Installation

前言

公司需要对学员提交的代码和任务进行统一的处理

微软的DevOps平台无疑是最好的解决方案

正好18年底微软发布了最新的2019版本

并且从TeamFoundationServer更名为DevOps Server

环境准备

  • Azure VM with Windows Server 2019 Datacenter
  • DevOps Server Installation package
  • SSL Certificate
  • SQL database

安装SQL Server

DevOps Server根据不同的应用场景，提供不同的部署方案，本次部署因为规模小而且主要作为测试用，所以采用SQL+DevOps单机安装的方案。具体需求参考文章

SQL安装简单，基本可以省略，下载地址

ps：记得开放SQL的通信端口

安装DevOps Server 2019 RC2

直接双击安装包，等待下载完成即可(磁盘空间≥4GB)

下载安装完成之后，即可开始配置

选择启动向导

点击这是新的Azure DevOps Server部署

选择高级部署，语言默认，中文就好

数据库设置这里，SQL Server实例写上你的服务器名\数据库实例名

eg:devops-server\tfs

账户这里我使用的默认服务账户即可

应用层选项这里，选为默认组，启用默认22端口勾选即可

搜索服务这里，默认选择即可，服务用户账户写你的管理员账号和密码即可

因为是单机部署，所以报表功能可以省去

创建一个你的团队项目集合,这里可默认，后期可以修改

检查无误后即可开始安装

这里会检查到java没有安装，因为搜索服务需要用到JRE，勾选安装即可

开始配置，等待10分钟左右

完成安装之后，即可使用浏览器访问http://devops-server访问

配置External Access & SSL

在上一步中，我们可以使用http://devops-server访问了

但是在前言中提到了，我们需要给学生使用，只能内网访问是不可能的，我们需要给它配置外网访问权限。

在所有程序中定位到Azure DevOps Server管理控制台

点击左侧应用层

找到右侧的更改公共URL

用你最后想用的公网域名替换掉原来的http://devops-server

再定位到IIS管理器，点开我们的网站，找到右侧的绑定

点击新建，协议选择https,端口号默认443，主机名写上你刚才的域名，指定给该域名的证书即可

最后打开你的域名解析，新建一个A记录，比如我这里子域名为devops，指向地址为服务器的ip

浏览器打开网址https://devops.yourdomain.com，已经可以访问
